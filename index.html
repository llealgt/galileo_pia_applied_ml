¿Qué es Machine Learning?

Definición: Arthur Samuel (1959)

"Field of study that gives computers the ability to learn without being explicitly programmed."
— Arthur Samuel, 1959

Traducción: Campo de estudio que da a las computadoras la capacidad de aprender sin ser programadas explícitamente.

Samuel creó un programa de damas (checkers). Después de miles de partidas, el programa se volvió mejor que su creador.

Punto clave: no se programa cada regla, el sistema aprende de la experiencia. Pregunta al público: si solo pudiera jugar 10 partidas en vez de miles, sería mejor o peor?

Definición: Tom Mitchell (1998)

"A computer program is said to learn from experience E with respect to some task T and performance measure P, if its performance at T, as measured by P, improves with experience E."

Ejemplo: ¿Es un gato?

E (Experiencia) Miles de imágenes etiquetadas "cat" / "not cat"
T (Tarea) Clasificar si una imagen nueva es gato o no
P (Desempeño) Cat → Cat : 80% de accuracy
Esta es la definición formal mas utilizada en ML. La clave: el desempeño MEJORA con más experiencia (datos).

Necesitamos una forma de medir el error → Función de Costo

Función de Costo

Midiendo qué tan bueno es nuestro modelo

Intuición: Midiendo el Error

Las líneas punteadas muestran el error (residual) de cada predicción
error(i) = f(x(i)) - y(i)

Queremos minimizar estos errores

Función de Costo: MSE

J(w,b) = \frac{1}{2m} \sum_{i=0}^{m-1} \left( f_{w,b}(x^{(i)}) - y^{(i)} \right)^2

Paso 1: Calcular el error: f_{w,b}(x^{(i)}) - y^{(i)}

Paso 2: Elevarlo al cuadrado: \left(\ldots\right)^2

  • Errores arriba de la recta son positivos, abajo son negativos — se cancelarían al sumar
  • Al elevar al cuadrado, todos se vuelven positivos
  • Además, penaliza errores grandes más que errores pequeños

Paso 3: Sumar sobre todos los ejemplos: \sum_{i=0}^{m-1}

Paso 4: Promediar: \frac{1}{2m} — el 2 facilita la derivada

Mean Squared Error (MSE) dividido por 2. El factor 1/2 es por conveniencia: al derivar, el 2 se cancela.

Objetivo Formal del ML

Con las piezas que ya tenemos, podemos definir el objetivo matemático del proceso de ML:

Dada una función de costo J, minimizar J en función de los parámetros w, b.

\underset{w,b}{\text{minimize}} \; J(w,b)

Se lee: "minimizar J respecto de w y b"

Los valores óptimos: w^*, b^* = \underset{w,b}{\text{argmin}} \; J(w,b)

Es decir, los valores de w y b que producen el menor error posible

Caso Simplificado: b = 0

Para entender la relación entre la hipótesis y el costo, asumimos b = 0:

Hipótesis simplificada:

f_w(x) = wx

Una recta que pasa por el origen

Función de costo (1 variable):

J(w) = \frac{1}{2m} \sum_{i=1}^{m} \left( f_w(x^{(i)}) - y^{(i)} \right)^2

Diferentes valores de w generan diferentes rectas, cada una con un costo diferente.

Objetivo:

\underset{w}{\text{minimizar}} \; J(w)

Encontrar el w que hace que la pendiente de la recta produzca el menor costo

Repaso: Cálculo Multivariable

Lo que necesitamos para entender Gradient Descent

Recordatorio: La Derivada

La derivada f'(x) mide la tasa de cambio de f en el punto x

f'(x) = \lim_{h \to 0} \frac{f(x+h) - f(x)}{h}

Es la pendiente de la recta tangente

Ejemplo: f(x)=x^2 \Rightarrow f'(x)=2x

Máximos y mínimos locales vs globales: donde f'(x) = 0

Derivada Parcial: Intuición

Cuando f depende de varias variables, la derivada parcial mide el cambio en una sola dirección, manteniendo las demás constantes.

\frac{\partial f}{\partial x}: variamos x, fijamos y
\frac{\partial f}{\partial y}: variamos y, fijamos x
Cortes en cada dirección producen curvas con pendientes diferentes

Derivadas Parciales: Ejemplos

Ejemplo 1: f(x,y) = x^2 + y^2

\frac{\partial f}{\partial x} = 2x   \frac{\partial f}{\partial y} = 2y
En (1, 2): ∂f/∂x = 2, ∂f/∂y = 4 → crece más rápido en dirección y

Ejemplo 2: f(x,y) = 3x^2y + 2xy^3

\frac{\partial f}{\partial x} = 6xy + 2y^3   \frac{\partial f}{\partial y} = 3x^2 + 6xy^2
Regla: derivar respecto a una variable, tratar la otra como constante

Ejemplo 3 (ML): J(w,b) = \frac{1}{2}(wx+b-y)^2

\frac{\partial J}{\partial w} = (wx+b-y) \cdot x   \frac{\partial J}{\partial b} = (wx+b-y)
Regla de la cadena: derivar el exterior, multiplicar por la derivada interior

Regla de la Cadena

Esencial para derivar funciones compuestas (como la función de costo):

\text{Si } h(x) = g(f(x)), \text{ entonces } h'(x) = g'(f(x)) \cdot f'(x)

Ejemplo paso a paso:

J(w) = \frac{1}{2}(wx - y)^2

Sea u = wx - y, entonces J = \frac{1}{2}u^2

\frac{dJ}{du} = u = (wx-y)   \frac{du}{dw} = x

\frac{dJ}{dw} = \frac{dJ}{du} \cdot \frac{du}{dw} = (wx - y) \cdot x

Esta es exactamente la fórmula que usaremos en Gradient Descent

El Vector Gradiente

\nabla f = \left[ \frac{\partial f}{\partial x}, \frac{\partial f}{\partial y} \right]

El gradiente es un vector que combina todas las derivadas parciales.

Ejemplo: f(x,y) = x^2 + y^2

\nabla f = [2x, 2y]
En (1, 0): \nabla f = [2, 0] → dirección +x
En (0, 1): \nabla f = [0, 2] → dirección +y
En (1, 1): \nabla f = [2, 2] → diagonal
En (0, 0): \nabla f = [0, 0] → mínimo!

Las flechas muestran \nabla f: apuntan desde el mínimo hacia afuera

Visualización del Gradiente

Cada flecha es el gradiente \nabla f en ese punto

  • Apuntan hacia afuera del mínimo (centro)
  • Son más largas lejos del mínimo (mayor pendiente)
  • Son cero en el mínimo
El negativo del gradiente
-\nabla f apunta hacia el mínimo = descenso más rápido

¿Por qué el Gradiente Apunta al Máximo Crecimiento?

La derivada direccional en dirección \mathbf{u} es:

D_{\mathbf{u}} f = \nabla f \cdot \mathbf{u} = |\nabla f| \cdot |\mathbf{u}| \cdot \cos(\theta)

\cos(\theta) se maximiza cuando \theta = 0

Es decir, cuando \mathbf{u} apunta en la misma dirección que \nabla f

Por lo tanto, \nabla f siempre indica la dirección donde f crece más rápido

Gradiente = Dirección de Máximo Incremento

El vector \nabla f(a,b) apunta en la dirección donde f crece más rápido desde el punto (a,b)

Derivada Direccional: Ejemplo

Sea f(x,y) = x^2 + y^2, punto (1, 2). Gradiente: \nabla f(1,2) = [2, 4], |\nabla f| \approx 4.47

Gradient Descent usa la dirección de máximo descenso: -\nabla f

Curvas de Nivel y Gradiente

Las curvas de nivel conectan puntos con el mismo valor de f

  • El gradiente ∇f es siempre perpendicular a las curvas de nivel
  • Apunta hacia donde f crece más rápido
  • Su magnitud indica qué tan empinada es la superficie
  • -∇f apunta hacia el mínimo
Esto es Gradient Descent:
Seguir -\nabla J paso a paso hasta llegar al mínimo

Conectando con ML

En nuestro caso, la función a minimizar es J(w,b):

\nabla J = \left[ \frac{\partial J}{\partial w}, \frac{\partial J}{\partial b} \right]

∂J/∂w dice cuánto y en qué dirección mover w

∂J/∂b dice cuánto y en qué dirección mover b

Juntos, el gradiente nos guía al mínimo de J

Gradient Descent

Descenso de Gradiente: el algoritmo de optimización más importante

La Idea Intuitiva: Paso a Paso

Click en el mapa para elegir punto de inicio. Colores = altura de la montaña.

Paso: 0 Altura: — Click en mapa o "Dar Paso"

Eso es exactamente Gradient Descent: dirección = -\nabla f  |  tamaño del paso = \alpha \cdot |\nabla f|

El Algoritmo

w := w - \alpha \frac{\partial J(w,b)}{\partial w}
b := b - \alpha \frac{\partial J(w,b)}{\partial b}

Repetir hasta convergencia

α = learning rate (tasa de aprendizaje) — tamaño del paso

∂J/∂w = derivada parcial — la dirección del paso

Antes (1 feature):

f_{w,b}(x) = wx + b

f_{\vec{w},b}(\vec{x}) = w_1 x_1 + w_2 x_2 + \cdots + w_n x_n + b

\vec{w} = [w_1, w_2, \ldots, w_n]

\vec{x} = [x_1, x_2, \ldots, x_n]

\vec{w} \cdot \vec{x} = \sum_{j=1}^{n} w_j x_j

De J al Gradiente

El objetivo: encontrar \vec{w}, b que minimicen J(\vec{w}, b)

\min_{\vec{w},b} J(\vec{w},b)

Para eso necesitamos las derivadas parciales:

\frac{\partial J}{\partial w_j} \quad \text{para cada } j = 1, \ldots, n \quad \text{y} \quad \frac{\partial J}{\partial b}

Primero veamos vectorización para hacer estos cálculos eficientemente, luego aplicamos Gradient Descent.

Vectorización

Código más rápido y más limpio

¿Qué es Vectorización?

Sin vectorización (loop)


f = 0
for j in range(n):
    f = f + w[j] * x[j]
f = f + b

Operaciones secuenciales: una a la vez

Con vectorización (NumPy)


f = np.dot(w, x) + b

Una sola operación: todas en paralelo

Mismo resultado, pero np.dot es 100-1000x más rápido con datos grandes

¿Por qué es más rápido?

  • Hardware paralelo: CPUs modernas tienen instrucciones SIMD que procesan múltiples datos simultáneamente
  • NumPy usa librerías optimizadas en C/Fortran (BLAS, LAPACK)
  • GPUs: miles de núcleos pequeños — vectorización es esencial para deep learning

Gradient Descent Vectorizado

Con loops


for i in range(m):
    err = f(x[i]) - y[i]
    for j in range(n):
        dw[j] += err * x[i][j]
    db += err
w = w - alpha * dw / m
b = b - alpha * db / m

Vectorizado


err = X @ w + b - y
dw = (1/m) * X.T @ err
db = (1/m) * np.sum(err)
w = w - alpha * dw
b = b - alpha * db

El operador @ en NumPy es multiplicación de matrices (np.matmul)

Gradient Descent

para Múltiples Variables

Algoritmo Extendido

Misma idea, más parámetros:

\text{Repetir hasta converger:}
w_j := w_j - \alpha \frac{\partial J}{\partial w_j} \quad \text{para } j = 1, \ldots, n
b := b - \alpha \frac{\partial J}{\partial b}

Actualización simultánea de todos los w_j y b

Derivadas Parciales

\frac{\partial J}{\partial w_j} = \frac{1}{m} \sum_{i=1}^{m} \left( f_{\vec{w},b}(\vec{x}^{(i)}) - y^{(i)} \right) x_j^{(i)}
\frac{\partial J}{\partial b} = \frac{1}{m} \sum_{i=1}^{m} \left( f_{\vec{w},b}(\vec{x}^{(i)}) - y^{(i)} \right)

Nota: cada \frac{\partial J}{\partial w_j} se multiplica por x_j^{(i)} — la feature correspondiente

1 Variable vs n Variables

1 feature

w := w - \alpha \frac{1}{m}\sum(f - y)x
b := b - \alpha \frac{1}{m}\sum(f - y)

n features

w_j := w_j - \alpha \frac{1}{m}\sum(f - y)x_j^{(i)}
b := b - \alpha \frac{1}{m}\sum(f - y)

La regla de b es idéntica. La de w_j solo agrega el subíndice j.

3 Técnicas de Feature Scaling

1. Dividir por máximo:

x_j^{\text{scaled}} = \frac{x_j}{\max(x_j)}

Rango resultante: [0, 1]

2. Mean normalization:

x_j^{\text{scaled}} = \frac{x_j - \mu_j}{\max(x_j) - \min(x_j)}

Rango resultante: aprox. [-0.5, 0.5]

3. Z-score normalization:

x_j^{\text{scaled}} = \frac{x_j - \mu_j}{\sigma_j}

Rango resultante: media=0, desv. est.=1

K-Nearest Neighbors (KNN)

"Dime con quién andas y te diré quién eres"

Idea: Clasificar un punto nuevo basándose en la clase de sus K vecinos más cercanos.

Tipo: No paramétrico — no aprende pesos, usa los datos directamente.

No necesita entrenamiento explícito — el "modelo" son los datos mismos.

Algoritmo KNN: Paso a Paso

  1. Calcular distancias entre el punto nuevo y todos los puntos de entrenamiento
  2. Encontrar los K puntos más cercanos (vecinos)
  3. Voto mayoritario: la clase más frecuente entre los K vecinos gana

Distancia Euclidiana:

d(\vec{x}, \vec{x}') = \sqrt{\sum_{j=1}^{n}(x_j - x'_j)^2}

Regresión Logística: Motivación

Necesitamos un modelo que prediga probabilidades:

  • Output debe estar entre 0 y 1
  • Interpretable como P(y=1|\vec{x})
  • Debe aprender parámetros (paramétrico)
  • Debe ser diferenciable (para gradient descent)
Solución: Aplicar la función sigmoid a la salida lineal
f_{w,b}(\vec{x}) = g(\vec{w} \cdot \vec{x} + b)

La Función Sigmoid

g(z) = \frac{1}{1 + e^{-z}}

Propiedades:

  • g(0) = 0.5
  • g(z) \to 1 cuando z \to +\infty
  • g(z) \to 0 cuando z \to -\infty
  • 0 < g(z) < 1 siempre
  • g'(z) = g(z)(1 - g(z))

Modelo de Regresión Logística

f_{\vec{w},b}(\vec{x}) = g(\vec{w} \cdot \vec{x} + b) = \frac{1}{1 + e^{-(\vec{w} \cdot \vec{x} + b)}}

Interpretación:

f(\vec{x}) = P(y = 1 | \vec{x}; \vec{w}, b)

"La probabilidad de que y=1 dado x, con parámetros w, b"

Ejemplo: Si f(\vec{x}) = 0.7
→ 70% de probabilidad de tumor maligno
→ P(y=0) = 1 - 0.7 = 0.3

¿Por qué NO usar MSE?

Con regresión lineal usamos: J = \frac{1}{2m}\sum(f(x^{(i)}) - y^{(i)})^2

Problema: con sigmoid, MSE es no convexa!

Múltiples mínimos locales

Un solo mínimo global

Cross-Entropy Loss

Loss para un ejemplo:

L(f(x), y) = \begin{cases} -\log(f(x)) & \text{si } y = 1 \\ -\log(1 - f(x)) & \text{si } y = 0 \end{cases}

Intuición:

  • Si y=1 y predigo f(x) \approx 1 → loss ≈ 0 ✓
  • Si y=1 y predigo f(x) \approx 0 → loss → ∞ ✗
  • Si y=0 y predigo f(x) \approx 0 → loss ≈ 0 ✓
  • Si y=0 y predigo f(x) \approx 1 → loss → ∞ ✗

Mueve el slider para ver el costo según la predicción f(x)

Forma Unificada del Costo

Combinamos ambos casos en una sola ecuación:

L = -y \cdot \log(f(x)) - (1-y) \cdot \log(1-f(x))

• Si y=1: L = -\log(f(x)) ✓

• Si y=0: L = -\log(1-f(x)) ✓

Función de costo completa:

J(\vec{w},b) = -\frac{1}{m}\sum_{i=1}^{m}\left[y^{(i)}\log(f(x^{(i)})) + (1-y^{(i)})\log(1-f(x^{(i)}))\right]

Esta función es convexa → gradient descent encuentra el mínimo global.
Deriva de Maximum Likelihood Estimation (MLE).

Costo Logístico en Python

import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

def logistic_cost(X, y, w, b):
    """Cross-entropy cost function"""
    m = len(y)
    f = sigmoid(X @ w + b)

    # Evitar log(0) con clip
    f = np.clip(f, 1e-7, 1 - 1e-7)

    cost = -1/m * np.sum(
        y * np.log(f) + (1 - y) * np.log(1 - f)
    )
    return cost

# Ejemplo
X = np.array([[1],[2],[3],[4],[5],[6]])
y = np.array([0, 0, 0, 1, 1, 1])
w = np.array([1.0])
b = -3.0

print(f"J = {logistic_cost(X, y, w, b):.4f}")

Gradient Descent para Logística

El algoritmo de GD es idéntico en estructura al de regresión lineal:

w_j := w_j - \alpha \frac{\partial J}{\partial w_j}

b := b - \alpha \frac{\partial J}{\partial b}

Las derivadas:

\frac{\partial J}{\partial w_j} = \frac{1}{m}\sum_{i=1}^{m}(f(x^{(i)}) - y^{(i)}) \cdot x_j^{(i)}

\frac{\partial J}{\partial b} = \frac{1}{m}\sum_{i=1}^{m}(f(x^{(i)}) - y^{(i)})

¡Misma Fórmula, Distinto f(x)!

Regresión Lineal Regresión Logística
f(x) \vec{w} \cdot \vec{x} + b \frac{1}{1+e^{-(\vec{w} \cdot \vec{x}+b)}}
∂J/∂w_j \frac{1}{m}\sum(f(x^{(i)})-y^{(i)})x_j^{(i)} \frac{1}{m}\sum(f(x^{(i)})-y^{(i)})x_j^{(i)}
Costo J MSE Cross-Entropy
Update w_j := w_j - \alpha \cdot \frac{\partial J}{\partial w_j}

La fórmula del gradiente luce igual — pero f(x) es diferente, ¡eso lo cambia todo!

GD Logístico: Implementación

import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

def gradient_descent_logistic(X, y, w, b, alpha, iters):
    m = len(y)
    J_history = []

    for i in range(iters):
        # Predicciones
        f = sigmoid(X @ w + b)

        # Gradientes
        dw = (1/m) * (X.T @ (f - y))
        db = (1/m) * np.sum(f - y)

        # Update
        w = w - alpha * dw
        b = b - alpha * db

        # Guardar costo
        f = np.clip(f, 1e-7, 1-1e-7)
        cost = -1/m * np.sum(y*np.log(f) + (1-y)*np.log(1-f))
        J_history.append(cost)

    return w, b, J_history

Tips para GD Logístico

1. Feature Scaling
Igual de importante que en regresión lineal — GD converge más rápido con features normalizadas.
2. Learning Rate
Mismo principio: si J sube → α muy grande. Empezar con α=0.01.
3. Convergencia
Monitorear J — debe decrecer en cada iteración. Detenerse cuando cambio < ε.
4. En la práctica
Usar sklearn.linear_model.LogisticRegression — usa métodos optimizados (L-BFGS, no GD simple).

Bias vs Variance

Descomposición del error:

E[\text{Error}] = \text{Bias}^2 + \text{Variance} + \sigma^2_{\text{Bayes}}

Bias (Sesgo)

Error por suposiciones simplificadoras del modelo. Modelo demasiado simple → no captura el patrón real.

Variance (Varianza)

Error por sensibilidad a los datos de entrenamiento. Modelo demasiado complejo → cambia mucho con datos distintos.

Error de Bayes (σ²)

Ruido irreducible. Ni el modelo perfecto puede eliminarlo.

1. Más datos de entrenamiento
Más ejemplos → modelo generaliza mejor
2. Selección de features
Eliminar features irrelevantes o redundantes
3. Regularización
Penalizar pesos grandes → modelo más simple (mantener todas las features)

Regularización: la técnica más poderosa cuando no podemos obtener más datos

Regularización: La Idea

Intuición: Si los pesos w_j son pequeños → el modelo es más "simple"

Agregar penalización al costo:

J_{reg}(\vec{w},b) = J(\vec{w},b) + \frac{\lambda}{2m}\sum_{j=1}^{n} w_j^2

  • \lambda = parámetro de regularización (hiperparámetro)
  • \lambda = 0 → sin regularización (puede overfit)
  • \lambda grande → pesos ≈ 0 (puede underfit)
  • No se penaliza b (solo los w_j)

Efecto de λ: Widget

Observa cómo λ suaviza la frontera de decisión

0.10

GD con Regularización

Update para w_j:

w_j := w_j\left(1 - \alpha\frac{\lambda}{m}\right) - \alpha\frac{1}{m}\sum_{i=1}^{m}(f(x^{(i)})-y^{(i)})x_j^{(i)}

Update para b (sin regularización):

b := b - \alpha\frac{1}{m}\sum_{i=1}^{m}(f(x^{(i)})-y^{(i)})

Nota: El factor (1 - \alpha\frac{\lambda}{m}) "encoge" w_j ligeramente en cada paso.

Por eso regularización L2 también se llama weight decay.

Función Softmax

Generalización de sigmoid a K clases:

P(y = k | \vec{x}) = \frac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}}

donde z_k = \vec{w}_k \cdot \vec{x} + b_k para cada clase k.

Propiedades:

  • Todas las probabilidades suman 1: \sum_k P(y=k) = 1
  • Cada P(y=k) \in (0, 1)
  • Si K=2, softmax = sigmoid

Cross-Entropy Multi-clase

Función de costo:

J = -\frac{1}{m}\sum_{i=1}^{m}\sum_{k=1}^{K} y_k^{(i)} \log(\hat{y}_k^{(i)})

Con One-Hot Encoding: y = [0, 0, 1, 0, ...]

Solo la clase verdadera contribuye al costo (las demás son 0).

Ejemplo: Si clase real = 2, y = [0, 0, 1, 0]

J = -log(ŷ₂) → penaliza si la probabilidad de clase 2 es baja
