1Objetivo

Construir un sistema de clasificación multi-clase (mínimo 3 clases) entrenando, comparando y ensamblando tres familias de modelos distintas. El proyecto evalúa tu capacidad de:

  • Diseñar y registrar experimentos de forma sistemática y reproducible.
  • Comparar modelos con métricas consistentes (precision, recall, F1, accuracy).
  • Diagnosticar underfit / overfit comparando desempeño en train vs test.
  • Combinar modelos en un ensamble y razonar sobre el resultado.
Importante

Este es un problema de clasificación multi-clase, no binario. El dataset elegido debe tener al menos 3 clases en la variable objetivo.

3Preparación de datos

  1. Carga y exploración mínima (shape, distribución de clases, nulos).
  2. Limpieza: manejo de nulos, encoding de categóricas, escalado si el modelo lo requiere (la red neuronal típicamente sí; los árboles no).
  3. Split único train-test (sugerido 70/30 u 80/20), estratificado por clase (stratify=y).
Restricción obligatoria

Solo se permite train-test split. NO uses cross-validation (ni cross_val_score, ni GridSearchCV con CV, ni KFold). Todos los experimentos y la selección del mejor modelo se reportan sobre el mismo test split.

Usa una random_state fija para que el split sea reproducible.

4.1 — Experimentos por modelo

Por cada uno de los 3 modelos debes ejecutar al menos 5 experimentos, variando hiperparámetros y/o arquitectura. Ejemplos de qué variar:

  • Random Forest: n_estimators, max_depth, min_samples_leaf, max_features, criterion.
  • XGBoost: n_estimators, max_depth, learning_rate, subsample, colsample_bytree, reg_lambda.
  • Red Neuronal: número de capas, neuronas por capa, activación, learning_rate, batch_size, epochs, dropout/regularización.

Para cada experimento, calcula las métricas en train y en test.

Ejemplo · loop de experimentos + append a disco

Una función reutilizable que calcula todas las métricas en train y test y agrega cada fila al CSV (creando el encabezado solo la primera vez). Así puedes correr el notebook varias veces sin perder resultados:

import os
import pandas as pd
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import precision_score, recall_score, f1_score, accuracy_score

BITACORA = "bitacora_experimentos.csv"

def metricas(y_true, y_pred, sufijo):
    return {
        f"precision_{sufijo}": precision_score(y_true, y_pred, average="macro"),
        f"recall_{sufijo}":    recall_score(y_true, y_pred, average="macro"),
        f"f1_{sufijo}":        f1_score(y_true, y_pred, average="macro"),
        f"accuracy_{sufijo}":  accuracy_score(y_true, y_pred),
    }

def registrar(exp_id, modelo, hiperparams, arquitectura, model):
    fila = {"experimento_id": exp_id, "modelo": modelo,
            "hiperparametros": str(hiperparams), "arquitectura": arquitectura}
    fila.update(metricas(y_train, model.predict(X_train), "train"))   # métricas en TRAIN
    fila.update(metricas(y_test,  model.predict(X_test),  "test"))    # métricas en TEST
    # append: encabezado solo si el archivo aún no existe
    pd.DataFrame([fila]).to_csv(BITACORA, mode="a",
        header=not os.path.exists(BITACORA), index=False)

# cada dict = un experimento con distintos hiperparámetros
configs = [
    {"n_estimators": 100, "max_depth": 5},
    {"n_estimators": 200, "max_depth": 10},
    {"n_estimators": 300, "max_depth": None},
    {"n_estimators": 200, "max_depth": 10, "min_samples_leaf": 5},
    {"n_estimators": 400, "max_depth": 15, "max_features": "sqrt"},
]

for i, cfg in enumerate(configs, start=1):
    model = RandomForestClassifier(**cfg, random_state=42).fit(X_train, y_train)
    registrar(f"RF_{i:02d}", "random_forest", cfg, "", model)   # ← se agrega al CSV

Reutiliza registrar(...) para XGBoost y la red neuronal. Cada llamada agrega una fila al mismo archivo; al volver a correr el notebook, las filas nuevas se añaden a las existentes.

Sugerencia

Para evitar filas duplicadas entre corridas, usa experimento_id distintos en cada ejecución (incluye fecha/hora o un contador de corrida), o al final deduplica con df.drop_duplicates().

5Bitácora de experimentos (CSV)

Debes generar un archivo bitacora_experimentos.csv que registre todos los experimentos (los ≥15 en total: ≥5 por familia). Cada fila = un experimento.

Obligatorio · append a disco

Los resultados deben agregarse (append) al CSV en disco a medida que se ejecutan, no sobrescribirse. Así puedes hacer varias corridas en distintos momentos sin perder los resultados previos. En la práctica: si el archivo no existe, créalo con encabezado; si ya existe, agrega las filas nuevas sin volver a escribir el encabezado (modo append).

Obligatorio · métricas en train Y test

La bitácora debe guardar las cuatro métricas calculadas tanto en train como en test: precision, recall, F1 y accuracy → 8 columnas de métricas en total. Esto permite la comparación train vs test y el diagnóstico de underfit/overfit.

6Ensamble — majority voting con bootstrap

Construye un ensamble de tipo majority voting combinando los 3 mejores modelos (uno por familia, los seleccionados en 4.2).

  • El ensamble predice la clase por voto mayoritario entre los 3 modelos.
  • Cada modelo base se entrena sobre una muestra bootstrap (muestreo con reemplazo del train set). Generas 3 réplicas bootstrap del conjunto de entrenamiento y entrenas cada mejor-modelo en su propia muestra.
  • En caso de empate (3 clases distintas), define y documenta una regla de desempate (p. ej. la clase del modelo con mayor F1 individual).
Ejemplo · ensamble en código 
import numpy as np
from scipy.stats import mode

modelos = [mejor_rf, mejor_xgb, mejor_nn]   # los 3 ganadores
preds = []
for m in modelos:
    # muestra bootstrap: con reemplazo, mismo tamaño que el train
    idx = np.random.choice(len(X_train), size=len(X_train), replace=True)
    m.fit(X_train[idx], y_train[idx])
    preds.append(m.predict(X_test))

preds = np.vstack(preds)              # shape (3, n_test)
y_ensamble = mode(preds, axis=0).mode.ravel()   # voto mayoritario

Regla de desempate sugerida: si los 3 votan distinto, quédate con la predicción del modelo de mayor F1 individual.

Implementación

Manual (entrenar cada modelo en su bootstrap y combinar predicciones con scipy.stats.mode) o con utilidades, siempre que cumplas: bootstrap por modelo + voto mayoritario. Evalúa el ensamble final en train y en test.
